# Step 3 — Lint stack (split by block)

**Ships in the SAME PR as Step 2. Can be developed in parallel with Step 2 if worktrees are used.**

## Acceptance criteria

### Facts block gates

- [ ] `spec-validator` passes.
- [ ] `facts-only-lint` passes.
- [ ] `inference-bait-lint` passes.
- [ ] `numeric_softening` token ban passes (same gates as brain output).

### Speculative block gates

- [ ] `spec-validator` passes.
- [ ] New lint rule: trailing inline disclaimer present verbatim (`Speculative — based partly on inferred data. Double-check.`).
- [ ] New lint rule: any numeric value NOT in fact pack must be wrapped in hedging language (presence of softening token required — inverse of facts-block rule).

### Chart block validation

- [ ] Structural only: must be `{title: string, columns: string[], rows: cell[][]}` or null.

### Failure test

- [ ] Deliberately-malformed run (invented tenant name in facts block, unhedged inferred number in speculative block, missing disclaimer) is REJECTED.
- [ ] DB remains untouched on rejection.
- [ ] Same fact pack + grounded blob run cleanly passes all lint.
