# _ASSISTANT/ — Session Brief + Spec Cleanup

**Date:** 2026-06-27

## Problem

- 83 specs in `docs/superpowers/specs/` — most are shipped and just sitting there
- 20+ handoff docs, most resolved
- No automated cleanup — stale docs accumulate
- New builds don't consistently create checks, so there's no signal when something is done

## Goal

Every Claude session opens with a current brief of what's actually in flight. Shipped specs get
archived automatically. New builds wire a check from day one so the cleanup loop closes itself.

---

## What we're building

### 1. Session brief extension (session-kickoff.mjs)

Extend the existing `scripts/session-kickoff.mjs` with a new section that appears in the KICKOFF
block already surfaced at session start. No new hook needed — this script already runs.

Adds to the KICKOFF output:

```
Spec clutter : 83 specs total · 61 candidates for archive · run `node scripts/assistant-weekly.mjs` to clean
```

If `_ASSISTANT/TODAY.md` exists (written by the weekly script), also prints its content so every
session sees the same daily brief without re-running the Supabase query.

### 2. `_ASSISTANT/` folder (gitignored except CLEANED.md)

```
_ASSISTANT/
  TODAY.md        ← written by weekly script, read by session-kickoff
  CLEANED.md      ← committed; log of every archive move
```

`TODAY.md` is regenerated by `assistant-weekly.mjs` and read by the session-kickoff hook. All
sessions that day see the same file. It contains:
- Date + what's in-flight from build-queue.md ([~] items)
- Open check count + any overdue checks (due date < today)
- Last SESSION_LOG entry headline
- Spec clutter count

### 3. `scripts/assistant-first-run.mjs` (one-time)

Scans the full spec and handoff corpus, archives what's dead, writes CLEANED.md.

**Dead spec criteria (ALL must be true):**
- Spec is ≥ 14 days old (filename date prefix)
- The corresponding build-queue.md entry is marked `[x]` (or the spec is not mentioned in the queue at all and is ≥ 60 days old)
- No open check in the Supabase ledger references the spec's slug/path
- No git commit in the last 14 days touches the spec file

**Dead handoff criteria:**
- Filename contains `SHIPPED`, `DONE`, or `cleanup` — OR file is ≥ 21 days old
- No open check references it

**Archive destinations:**
- `docs/superpowers/specs/_archive/` — committed
- `docs/handoff/_archive/` — committed

**CLEANED.md entry format:**
```
2026-06-27  ARCHIVED  docs/superpowers/specs/2026-05-16-tool-research-personalization-design.md
            Reason: build-queue [x], no open check, 42 days old
```

### 4. `scripts/assistant-weekly.mjs` (re-runnable)

Same logic as first-run but incremental — only processes files not already in `_archive/`.
Also regenerates `_ASSISTANT/TODAY.md`.

Run manually or wire to an existing GHA on Mondays.

### 5. `scripts/new-build.mjs <slug> "<label>"` (new build helper)

When starting a new build:
```
node scripts/new-build.mjs zip-report-rebuild "Rich /r/zip-report page"
```

Creates:
1. `docs/superpowers/specs/YYYY-MM-DD-<slug>-design.md` with a stub
2. Opens a check via `scripts/check.mjs open <slug>_live_verify "<label> live-verify"`

This closes the loop: every spec is born with a check. When the check closes and the build-queue
marks it `[x]`, the weekly script archives the spec automatically.

## File layout

```
_ASSISTANT/          ← gitignored except CLEANED.md
  TODAY.md
  CLEANED.md         ← committed

docs/superpowers/specs/
  _archive/          ← committed; dead specs land here

docs/handoff/
  _archive/          ← committed; resolved handoffs land here

scripts/
  session-kickoff.mjs      ← extended (existing)
  assistant-first-run.mjs  ← new, one-time
  assistant-weekly.mjs     ← new, re-runnable
  new-build.mjs            ← new, per-build helper
```

## What this does NOT do

- No cloud agents, no new GHA workflows (wire to an existing Monday workflow later if desired)
- Does not touch `build-queue.md` — stays human-maintained
- Does not delete anything — archive only
- Does not auto-run on every session start (session-kickoff reads TODAY.md if it exists; you run
  assistant-weekly.mjs to refresh it)
