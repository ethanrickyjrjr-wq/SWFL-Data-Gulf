### Task 1: SNAP-0 — Fix selection suppressed by FactChip buttons

**Root cause (verified by code reading):** Every metric value in `DataRow`/`MetricValueCell` is wrapped in a `FactChip`, which renders as a `<button>`. In `use-highlight.ts:422-428`, `onMouseUp` checks `target?.closest('button, [role="tab"]')` and, if true, **clears the selection and returns without calling `snapshot()`**. This means ANY attempt to select a metric value (which is always a button) is silently suppressed before the popup ever runs.

The button check was designed to prevent the selection popup from appearing on a pure click (which the chip's own `onClick` already handles). But it also suppresses drag-selections that END on a button, breaking all metric selection on report pages.

**Why the fix works:** We narrow the button suppression to pure clicks only (collapsed selection). Drag-selections that end on a button still reach `snapshot()`. A pure click on a FactChip keeps the old behavior: selection is cleared and snapshot is skipped, so only the chip's `onClick → onActivate` fires.

**Files:**
- Modify: `lib/highlighter/use-highlight.ts:422-431` (the `onMouseUp` handler)

**Interfaces:**
- No interface changes — behavior change only inside `onMouseUp`.

- [ ] **Step 2: Apply the fix**

Replace lines 422-431 with:
```typescript
function onMouseUp(e: MouseEvent) {
  mouseIsDown = false;
  const target = e.target as Element | null;
  // Suppress snapshot on a pure CLICK on a button/tab — the chip's own onClick
  // handles it. A drag-selection that ends on a button (non-collapsed) should
  // still trigger the popup (SNAP-0 fix).
  if (target?.closest('button, [role="tab"]')) {
    const sel = window.getSelection();
    if (!sel || sel.isCollapsed) {
      sel?.removeAllRanges();
      return;
    }
    // Non-collapsed (drag-select) — fall through to snapshot.
  }
  if (timer) clearTimeout(timer);
  timer = setTimeout(snapshot, 10);
}
```

- [ ] **Step 2: Create the utility**

Create `lib/format-date.ts`:
```typescript
/**
 * Format a date string (ISO YYYY-MM-DD or YYYY-MM-DDTHH:mm:ssZ) as MM/DD/YYYY.
 * Returns the input unchanged if it can't be parsed.
 */
export function formatDisplayDate(dateStr: string | null | undefined): string {
  if (!dateStr) return "";
  const m = dateStr.match(/^(\d{4})-(\d{2})-(\d{2})/);
  if (!m) return dateStr;
  return `${m[2]}/${m[3]}/${m[1]}`;
}
```

"use client";
import { useFiler } from "@/lib/briefcase/file-routing";
import type { ProjectItem } from "@/lib/project/items";
import { useState } from "react";

export function AddChartToProject({ title, chartKey }: { title: string; chartKey: string }) {
  const { file } = useFiler();
  const [filed, setFiled] = useState(false);
  function add() {
    const item: ProjectItem = { kind: "chart", title, chartKey, addedAt: new Date().toISOString() };
    file(item);
    setFiled(true);
  }
  return (
    <button
      type="button"
      onClick={add}
      disabled={filed}
      className="text-xs text-[#0a8078] disabled:opacity-60"
    >
      {filed ? "Added ✓" : "Add to project"}
    </button>
  );
}
