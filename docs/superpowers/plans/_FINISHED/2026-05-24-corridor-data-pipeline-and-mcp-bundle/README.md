## What's still broken (this brief is about this)

The 26 regenerated rows have **null `permit_zscore`, `saturation_index`, `lat`, `lng`** on every row. The Fiverr scaffold's 12 rows had hand-curated mock values for these fields; the regen merge preserved nothing because Supabase `corridor_name` slugs don't match the old hand-authored IDs. Consequence: `CorridorMarketScatter` on `/embed/charts` currently renders **zero bubbles**.

The previous behavior (12 bubbles of fake data) was worse ŌĆö invented permit z-scores, invented saturation indices, invented lat/lng. The new behavior (zero bubbles) is honest but not useful. Closing the gap is a real architectural project, not a fixture patch.

## Diagnosis ŌĆö four structural issues with the old shape

1. **`refinery/out/permits-swfl.json` does not exist.** `permits-swfl.mts` computes per-corridor `corridor_cells` in memory during a pack run, but only emits aggregated scalars (`county_weighted_z`, `saturation_index`) to `brains/permits-swfl.md`. Per-corridor z-scores are never persisted to a queryable artifact.
2. **Thin-pipe violation.** Brain Factory Rule #1: "A downstream brain never reads an upstream's branches ŌĆö only its `--- OUTPUT ---` block." A render-layer fixture that reaches into permits-swfl's per-corridor cells violates this rule.
3. **Corridor-ID namespaces don't match.** `permits-swfl` uses centroid IDs from `refinery/__fixtures__/lee-corridor-centroids.sample.json` (4 entries: `us-41-fort-myers`, `daniels-pkwy`, `naples-airport-pulling-north`, `naples-airport-pulling-south`). `corridor_profiles` uses `corridor_name` slugs (26 entries: `us-41-cleveland-ave-fort-myers`, `daniels-pkwy-i-75-to-ben-hill-griffin`, etc). There is no clean join. Hand-authored aliasing is required.
4. **`permits-swfl` is Lee-only.** ~10 of 26 corridors are Collier and will **permanently** have null `permit_zscore` until a Collier permits pack exists. And `saturation_index` is a county-level scalar, not a per-corridor field ŌĆö the fixture's per-row `saturation_index` was always invented.

## The right fix ŌĆö three parts in one PR sequence

### Part A ŌĆö Split the fixture by source-of-truth domain

The current `fixtures/corridors.json` pretends to be a cross-domain join in a single file. That was always wrong: rent comes from cre-swfl's domain, permit z from permits-swfl's, centroids from a geometry table. Split into three:

- `fixtures/corridor-rents.json` ŌĆö owned by `corridor_profiles` (cre-swfl). Regenerated by `npm run fixtures:corridors`. **This file is the asking-rent embed's source.**
- `fixtures/corridor-permits.json` ŌĆö owned by `permits-swfl` per-corridor sidecar (Part B). Lee-only by structural reality.
- `fixtures/corridor-centroids.json` ŌĆö owned by a centroid lookup table (initially the expanded Lee fixture; Collier added when geometry source lands).

Move `app/embed/cards/asking-rent/page.tsx` to read `fixtures/corridor-rents.json`. The scatter chart on `/embed/charts` will compose at render time from all three files (or render explicit "no data" states for what's missing).

**Do not delete `fixtures/corridors.json` mid-PR** ŌĆö leave it until consumers are migrated, then delete in the same PR. No window where consumers are broken.

### Part B ŌĆö `permits-swfl` per-corridor sidecar + centroid expansion + Collier acknowledgment

1. **Add a Stage 4 sidecar emit to `permits-swfl.mts`.** Writes `fixtures/corridor-permits.json` alongside the brain `.md`. Shape: `[{ corridor_id, headline_z, n_current, last_refined_at }]` where `headline_z` is the weighted average of `commercial_new` + `commercial_alteration` buckets weighted by `n_current` (matches the brain's `county_weighted_z` formula at the per-corridor level). **This is NOT a thin-pipe violation** ŌĆö it's an explicit, named, deterministic artifact the pack publishes as part of its output contract.
2. **Expand `refinery/__fixtures__/lee-corridor-centroids.sample.json` from 4 ŌåÆ ~16 Lee corridor centroids** matching corridor_profiles. Rename to drop `.sample.` once it's the real list.
3. **Hand-author a `corridor_id` alias table** keyed by `corridor_name` slug ŌåÆ centroid `corridor_id`. Lives in `refinery/lib/corridor-aliases.mts`. Single source of truth for the cross-domain join.
4. **Update `refinery/tools/regen-corridor-fixture.mts`** to read `fixtures/corridor-permits.json` + `fixtures/corridor-centroids.json` and join via the alias table. Output: `fixtures/corridor-rents.json` (rent + vacancy + absorption only) + a separate `fixtures/corridor-joined.json` if the scatter chart needs a pre-joined view.
5. **Document the Collier gap explicitly** in `docs/data-coverage.md` (new): "Permit z-score and saturation index are Lee-County-only until a Collier permits pack ships. ~10 corridors render permit fields as null. Charts must handle null explicitly ŌĆö no mock data."

## DO NOT do

- **DO NOT mock per-corridor `saturation_index`.** It's a county-level scalar. If the scatter chart needs a per-row "saturation" signal, derive one from `headline_z` thresholds in code, name it differently, and document the math. Don't reuse the field name.
- **DO NOT shortcut the centroid join.** Auto-deriving aliases from string similarity will silently misassign permits to the wrong corridor. Hand-authored table + unit test.
- **DO NOT keep `fixtures/corridors.json` after consumers migrate.** Leaving the cross-domain file around invites future shortcuts.
- **DO NOT reach into `permits-swfl`'s `corridor_cells` from any consumer.** Only read the published sidecar artifact. Thin pipe is the rule.
- **DO NOT auto-run `npm run refinery permits-swfl`** unless you're explicitly closing the v3ŌåÆv4 rollback in the same step. The TODO at the top of `brains/permits-swfl.md` is load-bearing: run it once, intentionally, with the sidecar emit code in place.
