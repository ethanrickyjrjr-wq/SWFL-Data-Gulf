## Concerns

### 1. Chart Block Fails Across All 5 — Fix Required

Every chart included Miami, Orlando, Tampa or other external benchmarks that were never in the fact pack. Contract violation: the chart prompt says "use only fact-pack values." Web-cited peer comparisons belong in the speculative block, not the chart.

**Two-part fix:**

- Tighten chart-block lint to reject any row value not present in the fact pack
- Sharpen prompt: "Chart rows must be values from FACT_PACK only. Web-cited peer values go in the speculative block."
- For peer comparisons: compare corridor-to-corridor (Mercato vs. Coconut Point, not Mercato vs. Tampa metro office). Brokers want apples-to-apples in their own market.

### 2. Naming — Strip the Milepost Suffixes

| Current                                     | Should Be       |
| ------------------------------------------- | --------------- |
| Bonita Beach Rd (US-41 to Sanibel Causeway) | Bonita Beach Rd |
| Daniels Pkwy (I-75 to Ben Hill Griffin)     | Daniels Pkwy    |

Sanibel Causeway is past Fort Myers Beach — wrong geography. The street-range suffix adds friction for brokers who just say "Bonita Beach" or "Daniels."

### 3. I-75 Coverage Gap (Naples Side)

Daniels Pkwy is Lee County. A user asking "what's near Pine Ridge and I-75" is asking about Collier County (Naples). We need a corridor that covers that node or a routing layer that maps geographic queries to the right corridor. Daniels should not be the answer to a Naples/I-75 query.

### 4. Speculative Block — Too Cautious, Not Enough Prediction

Current pattern: _"Without a quarterly time series, we cannot confirm direction."_
That defeats the purpose of the speculative block.

Better pattern: _"Without a verified quarterly cap rate series, direction is most likely tracking toward 25–50 bps expansion [INFERENCE] given the 1.2 pp uptick in Collier unemployment and slowing absorption signal — direction confirmation requires the next quarterly snapshot."_

Still hedged. Still labeled inference. But gives the operator something to push back on.

### 5. Vacancy Date Transparency

The vacancy figures come from `corridor_profiles.vacancy_rate_pct` — a hand-entered snapshot tagged with `metrics_period` (e.g., "2026-Q1"). This is not a freshly-ingested quarterly time series. The model correctly labels it as 2026-Q1 because that's the period tag, but consumers should know it's a snapshot, not a rolling feed.

### 6. Freshness Token / "VINTAGE: OLDEST-2026-01" — UX Problem

"VINTAGE: OLDEST-2026-01" means nothing to a broker. Proposed UX:

- Replace the freshness token line with the wave logo icon
- Logo click → hover shows "Data from 2021–2026"
- Full click → sources page with dates, Gulf Data rows, web sources, everything
- `[INTERNAL]` citation tags → replace with wave logo icon inline
- The report page (briefcase) already explains the data; no need to stamp freshness in the prose

### 7. Citation Rendering — Step 5 Work

Citations are currently raw URLs in JSON. Planned: `[web-14]` becomes a clickable footnote, sources panel renders at the bottom of the answer. Not implemented — Step 5 rendering task.
