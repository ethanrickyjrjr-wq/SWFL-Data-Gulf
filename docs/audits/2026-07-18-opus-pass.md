## Confirmed findings (ranked)

1. **False lake-provenance badge on web-sourced answers** ‚Äî `lib/assistant/report-path.ts:159` +
   `app/ask/AskPage.tsx:166`. Fix #15 widened `answered` to
   `neededComponents.length === 0 || webSources.length > 0`; fix #7 gates the "Grounded in SWFL
   lake data" badge on `answered === true`. A figure the lake lacked that the web fallback filled
   now renders with a lake-grounding claim ‚Äî the exact false-provenance #7 was killing, and a
   four-lane violation (lane-3 data labeled lane-1). Second consequence: `webFallbackForAnswer`
   (web-fallback.ts:259) verifies ANY figure on a figure-shaped ask, with no linkage to the
   metric's named `need` components ‚Äî a tangential verified benchmark marks a genuine gap
   `answered:true`, so the tracked-data-request funnel undercounts. Right shape: a provenance
   field (lake/web/gap) on the done frame; badge keys off provenance === lake, gap-log keys off
   components, not off one overloaded boolean.

2. **Single-digit moat check is now near-vacuous** ‚Äî `lib/assistant/gap-fill.ts:147`. Fix #43
   replaced the hard `target.length < 2 ‚Üí false` guard with "must appear as its own number
   token." `valueAppearsInText` is THE shared moat check for web citations AND upload
   attribution (compose-chart upload points). Virtually any page or document contains a
   standalone digit ("Page 5", "July 5", "5 of 12"), so an invented single-digit figure ("5% cap
   rate") now verifies off incidental tokens and ships cited / attributed to the user's own
   upload. Simultaneously the motivating case still fails: "5.0%" tokenizes as `5.0` ‚Üí
   `digitsOf` = "50" ‚âÝ "5". Too loose and too strict at once. Right shape: numeric equality
   (`Number(token) === value`) for the short-target path, and consider requiring unit/context
   adjacency for bare single digits to keep the moat meaningful.

3. **Delta "appeared" branch fabricates "new activity" emails** ‚Äî `lib/email/activation/delta.ts:142`.
   Fix #73's first-appearance branch treats any deltable brain absent from `prev.lines` as a new
   signal. But snapshot lines are a RANKED, CAPPED selection (`selectDossierLines`, top-8
   headline slice) that also silently omits a brain whose build transiently failed. A brain
   rotating back into the top-8 (or recovering from a failed build) flips `has_change=true` and
   email #2 claims "new activity in your ZIP" with zero underlying change ‚Äî violating the
   contract stated in this same file ("never a fabricated change"), and the deliberate
   line-131 refusal of the symmetric disappearance case shows why. Right shape: only claim
   first-appearance when the brain was absent from the UNCAPPED candidate set, or carry a
   per-brain "had any line ever" marker in the snapshot.

4. **ReportFooter prints raw internal strings on public pages** ‚Äî `app/r/_components/report-shell.tsx:76`.
   Fix #9 renders `As of {asOfFromToken(freshnessToken) ?? freshnessToken}` ‚Äî the raw-token
   fallback this same commit removed elsewhere (#42, #46 both omit-when-unparseable). Live
   today: `app/r/should-i-sell/[zip]/page.tsx:196` passes `stress?.lastChecked` ‚Äî an ISO
   timestamp, not a token ‚Äî so that public footer renders the raw ISO string. Violates the
   locked MM/DD/YYYY / no-raw-token rule. Fix: omit the line when unparseable, and have
   should-i-sell pass a formatted date.

5. **Currency-delta fix misses the second currency metric that already exists** ‚Äî
   `lib/email/grounded-report.ts:114`. `CURRENCY_METRIC_KEYS` holds only
   `housing.median_sale_price`; `env.flood_aal_usd` (snapshot.ts:131) flows through the same
   delta pipeline (the golden `cases.ts` even includes it as a delta row). Its "What changed"
   row renders `30,074` bare while the key-figures section shows `$30,074 / yr` ‚Äî the exact #89
   defect, still live. Worse: the regenerated `delta-changed.html` golden now LOCKS the wrong
   output as expected. The fix log's "the one currency delta cell that exists today" is
   factually wrong. Right shape: thread `format`/prefix through `SnapshotMetric`/`MetricChange`
   from HOUSING_CELLS' `format:"currency"` (the one authority) instead of a renderer-side set.

6. **og-image SSRF hardening ‚Äî the requested security read: insufficient as shipped** ‚Äî
   `lib/email/og-image.ts:131`. The per-hop loop re-validates each redirect against
   `isSafePublicUrl` ‚Äî hostname-pattern checks only, no DNS resolution. `lib/email/safe-fetch.ts`
   (same directory) exists precisely because of this gap ‚Äî its header names og-image's
   weaknesses and provides `lookup()`-then-`isPrivateOrReservedIp`. A hostname whose A-record
   points at 169.254.169.254 passes every hop check. Secondary: the loop never consumes/cancels
   3xx response bodies (nor the final body on early-return paths) ‚Äî in undici each unread body
   pins its socket until GC; up to 5 per call, multiplied by resolveHeroPhoto's 4 candidate URLs
   per email fill. Fix: rebuild on safe-fetch's resolve-then-check (extend it with a
   validated-hop mode), and `void res.body?.cancel()` on every non-consuming path.

7. **ZipChoropleth error state is permanent** ‚Äî `components/charts/ZipChoropleth.tsx:98`.
   Fix #50's error branch replaces the ref div with `<ChartError />`, so after one failure
   `ref.current` is null and the effect's `if (!container) return` exits before the
   `setLoadError(false)` reset ‚Äî no deps change can ever recover; only a full remount. One
   transient blip on `/maps/lee-collier.svg` kills the chart for the page's life. Fix: keep the
   ref div mounted (overlay the error) or gate the early-return on the error state.

8. **MediaPanel rollback misses thrown fetches** ‚Äî `components/email-lab/MediaPanel.tsx:85`.
   Fix #52 reverts on `!res.ok`, but `await fetch` with no try/catch means a network-level
   failure (offline, server unreachable) skips rollback AND the error message ‚Äî optimistic
   rename/delete sticks, silently reverting on next reload: the same silent-failure class #52
   targeted, plus an unhandled rejection. Same in `remove()` at :100. Fix: wrap both in
   try/catch that routes to the same revert+error path.

9. **`PROPERTY_TYPE: "Single Family"` survived the invented-defaults sweep** ‚Äî
   `lib/email/templates/token-defaults.ts:118`. Fix #20 converted every per-property fact to
   "‚Äî" under the comment "never an invented fact" ‚Äî except this one. A condo rendered with
   defaults claims "Single Family" with full confidence while every neighboring field reads "‚Äî".
   One-line fix to "‚Äî".

10. **render-golden.test.ts now misstates its own evidence base** ‚Äî
    `lib/email/render-golden.test.ts:22`. eb071f90 regenerated 9 of 10 goldens from CURRENT
    output, but the docstring and describe label still claim a frozen pre-spine reference
    "captured @91b6aec" / "NON-CIRCULAR." The fix log admits the test is now self-referential ‚Äî
    but only in the audit doc, not where the next maintainer reads. Update the header to state
